# Network Intrusion Detection using Machine Learning on CIC-IDS2017

This repository contains the full implementation for a multiclass network
intrusion detection system, built and evaluated on the CIC-IDS2017 dataset.


## Dataset

This project uses seven CSV files from the **MachineLearningCVE** version of
the CIC-IDS2017 dataset, covering Tuesday, Wednesday, both Thursday files,
and all three Friday files. Monday's file is excluded, as it contains only
benign traffic and introduces no additional classes.

The raw dataset is not included in this repository, since it is a large,
publicly available research dataset with a permanent official source.
To reproduce this project, download it directly from the source below.

**Download the dataset here:**
https://www.unb.ca/cic/datasets/ids-2017.html

From the downloaded `MachineLearningCVE` folder, place the following seven
files into a `data/` folder in the root of this repository:

```
Tuesday-WorkingHours.pcap_ISCX.csv
Wednesday-workingHours.pcap_ISCX.csv
Thursday-WorkingHours-Morning-WebAttacks.pcap_ISCX.csv
Thursday-WorkingHours-Afternoon-Infilteration.pcap_ISCX.csv
Friday-WorkingHours-Morning.pcap_ISCX.csv
Friday-WorkingHours-Afternoon-PortScan.pcap_ISCX.csv
Friday-WorkingHours-Afternoon-DDos.pcap_ISCX.csv
```

The dataset was originally published by the Canadian Institute for
Cybersecurity:

> Sharafaldin, I., Lashkari, A.H. and Ghorbani, A.A. (2018) Toward generating
> a new intrusion detection dataset and intrusion traffic characterization.
> *Proceedings of the 4th International Conference on Information Systems
> Security and Privacy (ICISSP 2018)*, pp. 108-116.

## Setup Instructions

1. Clone this repository:
   ```
   git clone <repository-url>
   cd <repository-name>
   ```

2. Create a Python environment and install dependencies:
   ```
   pip install pandas numpy scikit-learn xgboost matplotlib seaborn joblib jupyter
   ```

3. Download the dataset as described above and place the seven CSV files
   into a `data/` folder in the repository root.

4. Run `notebooks/01_data_preparation.ipynb` from top to bottom. This will:
   - Load and combine the seven CSV files
   - Perform exploratory data analysis
   - Clean the data
   - Split and scale the data
   - Save processed arrays to `processed_data/` (created automatically)

5. Run `notebooks/02_modelling_and_evaluation.ipynb` from top to bottom. This will:
   - Load the processed data generated in step 4
   - Train and compare a Random Forest model and an XGBoost model
   - Tune the XGBoost model via randomised hyperparameter search
   - Evaluate the final tuned model on the held-out test set
   - Save the final model, scaler, and label encoder to `saved_model/`

## Reproducing Results

The final tuned XGBoost model achieved the following performance on the
held-out test set:

| Metric | Score |
|---|---|
| Accuracy | 99.84% |
| Macro F1 | 0.914 |
| Weighted F1 | 0.998 |

Full per-class results are available in `results/test_set_results_per_class.csv`
and in the report.

## Notes

- `data/` and `processed_data/` are intentionally not committed to this
  repository. `data/` must be downloaded manually as described above, and
  `processed_data/` is fully regenerated by running notebook 01 on that data.
- The trained model and supporting objects in `saved_model/` **are** committed,
  since they are required to reproduce the demonstration in notebook 02
  without retraining from scratch.
